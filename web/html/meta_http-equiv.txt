？1.<meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1" />

解：1,meta属性只能放在head头部中
	2,"IE = edge, chrome = 1"表示IE浏览器采用最高渲染方式渲染页面，chrome = 1表示启用chrome frame
	3, http-equiv="X-UA-Compatible" X-UA-Compatible是自从IE8新加的一个设置，对于IE8以下的浏览器是不识别的。
	通过在meta中设置X-UA-Compatible的值，可以指定网页的兼容性模式设置。

?2.<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1"/>

解：1, 这条命令来完成自适应网页设计的目标。同样放在网页代码的头部。
	2,viewport是网页默认的宽度和高度，上面这行代码的意思是，网页宽度默认等于屏幕宽度（width=device-width），原始缩放比例（initial-scale=1）为1.0，即网页初始大小占屏幕面积的100%，maximum-scale = 1，表示最大缩放比例为1（100%）。

?3.	<meta http-equiv="Cache-Control" content="no-transform" />
	<meta http-equiv="Cache-Control" content="no-siteapp" />
	
解：禁止百度转码。（
					1. web页面转码的意义
 互联网上融合了WEB和WAP两类网页资源，WEB网页资源占比远多于WAP网页，但目前绝大多数移动终端的浏览器都不能直接浏览WEB网页。因此，为确保WAP可用性，要对
WAP搜索结果中的WEB网页进行格式转换，转换为WAP网页，使其能在移动终端浏览器被浏览。百度的转码技术即是为实现此目的、服务于WAP搜索的附属产品，各大搜
索引擎均拥有并提供类似转码技术.	

					2. 百度转码的策略？
   为了最大程度解决WEB页面在手机上的可用性，转码须去除WEB页面中不能在手机浏览器上浏览的内容，和无法用WAP描述语言描述的部分内容，并将网页布局由2维布
局配为1维布局。）
